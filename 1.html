시그널의 개념

- 프로세스에 뭔가 발생했음을 알리는 간단한 메시지를 비동기적으로 보내는 것
- 메시지는 무엇이 발생했는지를 표시하는 미리 정의된 상수를 사용

시그널의 발생

- 시그널은 소프트웨어 인터럽트 (주로 시스템콜을 사용하는 OS에서 정의한 API)
  - 나누기0 과같은 프로그램에서 예외적인 상황이 일어나는 경우
  - 프로세스가 kill 함수와 같이 시그널을 보낼 수 있는 함수를 사용해 다른 프로세스에 시그널을 보내는 경우
  - 사용자가 Ctrl+C 같은 인터럽트 키를 입력하는 경우

시그널 처리 방법

- 프로세스가 받은 시그널에 따라 기본동작을 수행 각 시그널에는 기본동작이 지정되어 있음
  - 대부분 시그널의 기본동작은 프로세스를 종료
  - 시그널을 무시하거나 프로세스의 수행 일시 중지(Suspend)/재시작(Resume) 등을 기본동작으로 수행
- 프로세스가 받는 시그널을 무시 (프로세스가 시그널을 무시하기로 지정하면 유닉스는 프로세스에 시그널을 전달하지 않음)
- 프로세스는 시그널의 처리를 위해 미리 함수를 지정(Signal Handler)해놓고 시그널을 받으면 해당 함수를 호출 (동작이 완료되면 기존 처리작업을 계속 수행)
- 프로세스는 특정 부분이 실행되는 동안 시그널이 발생하지 않도록 블록 할 수 있음(블록된 시그널은 큐에 쌓여 있다가 시그널 블록이 해제되면 전달)

시그널 보내기

- 시그널 보내는 함수
  - kill()
  - raise()
  - abort()

- PID가 3255인 프로세스를 강제 종료하는 명령

# kill -9 3255 // kill SIGKILL 3255

시그널 보내기 함수

- 시그널 보내기 : kill(2)

#include < sys/types.h > #include < signal.h > int kill (pid_t pid, int sig); // pid : 시그널을 받을 프로세스 ID // sig : pid로 지정한 프로세스에 보내려는 시그널

- kill 함수는 pid에 대응하는 프로세스에 sig로 지정한 시그널을 보냄
  - pid는 특정 프로세스 또는 프로세스 그룹을 의미
  - sig에 0(널 시그널)을 지정하면 실제로 시그널을 보내지 않고 오류를 확인 (pid가 정상인지 검사)
- pid에 지정한 값에 따라 시그널을 어떻게 보낼 것인지를 결정
  - pid 가 0 보다 큰 수 : pid로 지정한 프로세스에게 시그널을 보냄
  - pid 가 -1 이 아닌 음수 : 프로세스 그룹 ID가 pid의 절대값인 프로세스 그룹에 속하고 시그널을 보낼 권한을 가지고 있는 모든 프로세스에게 시그널을 보냄
  - pid 가 0 : 특별한 프로세스를 제외하고 프로세스 그룹ID가 시그널을 보내는 프로세스의 프로세스 그룹 ID와 같은 모든 프로세스에게 시그널을 보냄
  - pid 가 -1 : 시그널을 보내는 프로세스의 유효 사용자 ID가 root가 아니면 특별한 프로세스를 제외하고 프로세스의 실제 사용자 ID가 시그널을 보내는 프로세스의 유효 사용자 ID와 같은 모든 프로세스에게 시그널 보냄

- 시그널 보내기 : raise(3)

#include < signal.h > int raise (int sig); // sig : 보내려는 시그널 번호

- 호출한 프로세스에 인자로 지정한 시그널을 보냄
  - 핸들러가 호출되면 시그널 핸들러의 수행이 끝날 때ㅔ까지 raise 함수는 리턴하지 않음
  - 수행을 성공하면 0, 실패하면 -1을 리턴

- 시그널 보내기 : abort(3)

#include < stdlib.h > void abort (void);

- 이를 호출한 프로세스에 SIGABRT 시그널을 보냄
  - 프로세스를 비정상적으로 종료시키고 코어 덤프 파일 생성
  - 최소한 해당 프로세스가 연 파일은 모두 닫음
- 프로세스를 종료시키므로 리턴하지 않음

시그널 기본처리

- 종료하기전에 처리할 작업이 남아 있거나 특정 시그널에 대해 종료하고 싶지 않을 때 시그널 호출 시 수행할 함수를 지정(시그널 핸들러)
- 시그널을 확인해 처리하는 일을 시그널 붙잡기(catching a signal)

시그널 핸들러 함수

- 시그널 핸들러 지정 : signal(3)

#include < signal.h > void (*signal (int sig, void (*disp)(int))) (int); // sig : 시그널 핸들러로 처리하려는 시그널 // disp : 시그널 핸들러의 함수명

- 해당 시그널을 처리할 함수나 상수를 지정
  - sig : SIGKILL, SIGSTOP을 제외한 모든 시그널을 지정할 수 있음
  - disp : sig로 지정한 시그널을 받았을 때 처리할 방법을 지정
    - 시그널 핸들러 주소
    - SIG_IGN : 시그널을 무시하도록 지정
    - SIG_DFL : 시그널의 기본 처리 방법을 수행하도록 지정
- 시그널 핸들러의 주소를 리턴
  - 실패하면 SIG_ERR
- disp가 함수 주소, sig가 SIGKILL, SIGTRAP, SIGPWR이 아니면 signal 함수는 시그널을 처리한 후 시그널 처리방법을 기본처리 방법으로 재설정(SIG_DFL)
  - 시그널 처리를 계속하려면 signal 함수를 호출해 시그널을 처리한 후 다시 signal 함수를 설정해야 함

- 시그널 핸들러 지정 : sigset(3)

#include < signal.h > void (*sigset (int sig, void (*disp)(int))) (int); // sig : 시그널 핸들러로 처리하려는 시그널 // disp : 시그널 핸들러의 함수명

- sigset과 signal 함수의 인자구조는 동일
  - sig : SIGKILL, SIGSTOP을 제외한 모든 시그널을 지정할 수 있음
  - disp : sig로 지정한 시그널을 받았을 때 처리할 방법을 지정
    - 시그널 핸들러 주소
    - SIG_IGN : 시그널을 무시하도록 지정
    - SIG_DFL : 시그널의 기본 처리 방법을 수행하도록 지정
- 시그널 핸들러의 주소를 리턴
  - 실패하면 SIG_ERR
- 시그널 핸들러가 한 번 호출된 후에 기본동작으로 재설정하지 않고 시그널 핸들러를 자동으로 재지정
  - 시그널 핸들러를 재지정 할 필요가 없음

시그널 집합

- POSIX 표준에서 시그널과 관련해 정의한 함수 중 상당 수는 시그널을 개별적으로 처리하지 않고 복수의 시그널을 처리할 수 있게 함 (시그널 집합(signal set))

시그널 집합의 개념

- 시그널 집합은 비트 마스크로 표현 (시그널 하나가 비트 하나를 가리킴)
- 유닉스에서 시그널 집합의 처리를 위해 sigset_t이라는 구조체를 제공
  - sigset_t 구조체는 크기가 4인 unsigned int 배열을 사용

typedef struct { unsigned int __sigbits[4]; } sigset_t;

시그널 집합 처리 함수

- 시그널 집합 비우기 : sigemptyset(3)

#include < signal.h > int sigemptyset(sigset_t *set); // set : 비우려는 시그널 집합의 주소

- 시스템에서 정의한 모든 시그널을 배제해 인자로 지정한 시그널 집합을 빈 집합으로 만듬 (시그널 비트를 모두 0으로 설정)
- 수행을 성공하면 0, 실패하면 -1 리턴

- 시그널 집합에 모든 시그널 설정 : sigfillset(3)

#include < signal.h > int sigfillset(sigset_t *set); // set : 설정하려는 시그널 집합의 주소

- 인자로 받은 시그널 집합을 시스템에서 정의한 모든 시그널을 포함하는 집합으로 만듬 (시그널 집합의 모든 비트를 1로 설정)
- 수행을 성공하면 0, 실패하면 -1 리턴

- 시그널 집합에 시그널 설정 추가 : sigaddset(3)

#include < signal.h > int sigaddset (sigset_t *set, int signo); // set : 시그널을 추가하려는 시그널 집합의 주소 // signo : 시그널 집합에 추가로 설정하려는 시그널

- signo로 지정한 시그널을 set으로 지정한 시그널 집합에 추가
- 수행을 성공하면 0, 실패하면 -1 리턴

- 시그널 집합에서 시그널 설정 삭제 : sigdelset(3)

#include < signal.h > int sigdelset (sigset_t *set, int signo); // set : 시그널을 삭제하려는 시그널 집합의 주소 // signo : 시그널 집합에서 삭제하려는 시그널

- signo로 지정한 시그널을 set으로 지정한 시그널 집합에서 제거
- 수행을 성공하면 0, 실패하면 -1 리턴

- 시그널 집합에 설정된 시그널 확인 : sigismember(3)

#include < signal.h > int sigismember (sigset_t *set, int signo); // set : 확인하려는 시그널 집합의 주소 // signo : 시그널 집합에 설정되었는지 확인하려는 시그널

- signo로 지정한 시그널이 set으로 지정한 시그널 집합에 포함되어 있으면 1, 없으면 0 리턴

sigaction 함수의 활용

- signal, sigset 함수처럼 시그널을 받았을 때 이를 처리할 수 있는 함수를 지정

sigaction 구조체

- < sys/signal.h >에 정의
- 구조체 멤버
  - 시그널 처리를 위한 시그널 핸들러 주소
  - 시그널 핸들러가 수행하는 동안 블록될 시그널
  - 추가적인 기능을 설정할 수 있는 플래그

truct sigaction { int sa_flags; union { void (*sa_handler)(); void (*sa_sigaction)(int, siginfo_t *, void *); } _funcptr; sigset_t sa_mask; };

- sa_flag : 시그널 전달 방법을 수정할 플래그를 지정  
- sa_handler, sa_sigaction : sa_handler, sa_sigaction는 메모리가 중첩됨
    - sa_flags 에 SA_SIGINFO 설정되어 있지 않으면 sa_handler에는 시그널을 처리할 동작을 지정
    - sa_flags 에 SA_SIGINFO 가 설정되어 있으면 sa_sigaction 멤버를 사용
- sa_mask : 핸들러가 동작 중일 때 블록할 시그널을 시그널 집합으로 지정
  - 시그널 핸들러가 시작되어 시그널을 전달할 때 이미 블록된 시그널 집합에 sa_mask로 지정한 시그널 집합을 추가
  - sa_flags에 SA_NODEFER를 설정하지 않으면 시그널 핸들러를 호출하게 한 시그널도 블록

sigaction 함수

- 시그널을 받아 이를 처리할 시그널 핸들러를 지정할 뿐만 아니라 플래그를 설정해 시그널을 처리하는 과정 제어
- 시그널 핸들러가 수행되는 동안 다른 시그널을 블록할 수 있음

#include < signal.h > int sigaction (int sig, const struct sigaction *restrict act, struct sigaction *restrict oact); // sig : 처리할 시그널 // act : 시그널을 처리할 방법을 지정한 구조체 주소 // oact : 기존에 시그널을 처리하던 방법을 저장할 구조체 주소

- sig에 지정한 시그널을 받았을 때 처리할 방법을 sigaction 구조체인 act로 받음
  - SIGKILL, SIGSTOP를 제외한 어떤 시그널도 사용할 수 있음
  - 두 번째, 세 번째 인자는 sigaction 구조체를 사용해 지정 두 번째 인자로 NULL 또는 sig에 지정된 시그널을 받았을 때 처리할 방법을 지정한 구조체 주소를 지정
  - 세 번째 인자에는 NULL 또는 기존 처리방법이 저장
- 수행을 성공하면 0, 실패하면 -1 리턴

시그널 발생 원인 검색

- sa_flags 에 SA_SIGINFO 플래그를 지정하면 시그널이발생한 원인을 알 수 있음

void handler(int sig, siginfo_t *sip, ucontext_t *ucp); // sig : 시그널 핸들러를 호출할 시그널 // sip : 시그널이 발생한 원인을 담은 siginfo_t 구조체 포인터 // ucp : 시그널이 전달될 때 시그널을 받은 프로세스의 내부 상태를 나타내느 ucontext_t 구조체 포인터
typedef structure { int si_signo; int si_errno; int si_code; union sigval si_value; union { .... } __data; } siginfo_t;

- si_signo : 시스템에서 제공하는 시그널번호 저장
- si_errno : 0 또는 시그널과 관련된 오류 번호를 저장
- si_code : 시그널 발생원인을 정의하는 코드 저장
    - SI_NOINFO -> si_signo만 의미가 있고 siginfo_t 구조체의 나머지 멤버는 사용되지 않음
- __data : 시그널 종류에 따라 값이 저장 (세부내용은 < sys/siginfo.h > 참고)

- 사용자 프로세스에 의한 시그널발생 원인 코드(si_code 값)

- si_code 값이 양수면 시스템에서 시그널을 생성

man -s 3HEAD siginfo.h // 시그널별 발생원인을 나타낸 코드 확인 (< sys/machsig.h >에 정의되어 있음)

- 시그널 발생 원인 출력 : psiginfo(3)

#include < siginfo.h > void psiginfo (siginfo_t *pinfo, char *s); // pinfo : 시그널 발생 원인에 관한 정보를 저장하고 있는 구조체 포인터 // s : 출력할 문자열

- siginfo_t 구조체 포인터를 인자로 받아 시그널이 발생한 원인을 표준 오류로 출력
    - pinfo : 시그널 핸들러의 두번째 인자로 받은 siginfo_t 구조체의 주소를 지정
- 함수를 실행하면 s에 지정한 문자열 출력 후 시그널 정보 출력

알람 시그널

- 일정 시간 후에 시그널 발생

알람 시그널 함수

- 알람 시그널 생성 : alarm(2)

#include < unistd.h > unsigned int alarm(unsigned int sec); // sec : 알람을 발생시킬 때까지 남은 시간 (sec)

- sec로 지정한 시간이 지나면 SIGALRM 시그널이 생성되어 프로세스에 전달
- 프로세스별 알람은 하나만 설정할 수 있음
  - 알람 시그널을 생성하기 전에 다시 alarm 함수를 호출하면 이전 설정은 없어지고 재설정 됨
- 인자로 0을 지정하면 이전에 설정한 알람 요청을 모두 취소
- 이전에 호출한 alarm 함수의 시간이 남아있으면 해당시간, 그렇지 않으면 0을 리턴

인터벌 타이머

- 알람 시그널은 정해진 시간에 한 번 시그널을 보냄
- 일정 간격을 두고 타이머 역할
- 종류
    - ITIMER_REAL : 실제시간 사용. 타이머 만료시 SIGALRM 시그널 생성
    - ITIMER_VIRTUAL : 프로세스의 가상 시간 사용 (프로세스가 동작중일 때만 작동). 타이머 만료시 SIGVTALRM 시그널 생성
    - ITIMER_PROF : 프로세스를 위해 실행중인 시간과 프로세스의 가상 시간을 모두 사용. 타이머가 만료되면 SIGPROF 시그널 생성
    - ITIMER_REALPROF : 멀티스레드 프로그램의 실제 시간을 측정할 때 사용(실제시간 사용). SIGPROF

- 타이머 정보 검색 : getitimer(2)

#include < sys/time.h > int getitimer (int which, struct itimerval *value);

- which : 검색할 타이머의 종류 지정
- value : 타이머 정보를 저장할 구조체 포인터
- itimerval 구조체는 < sys/time.h > 파일에 정의되어 있음

struct itimerval { struct timeval it_interval; struct timerval it_value; };

- it_interval : 타이머 간격정보가 저장 (0이면 다음번 타이머가 만료될 때 타이머 기능이 멈춤)
- it_value : 만료때까지 남은 시간 저장 (0이면 타이머 기능이 멈춤)
- 이러한 시간정보들은 timeval 구조체에 정의

struct timeval { time_t tv_sec; suseconds_t tv_usec; };

- 초와 마이크로 초 단위로 시간을 저장

- 타이머 설정 : setitimer(2)

#include < sys/time.h > int setitimer (int which, const struct itimerval *value, struct itimerval *ovalue);

- which : 설정할 타이머의 종류 지정
- value : 설정할 타이머 정보를 저장해 인터벌 타이머 설정
- ovalue : NULL 또는 이전 설정값을 저장
- value 와 ovalue는 itimerval 구조체를 가리키는 포인터

기타 시그널 처리 함수

시그널 정보 출력

- 시그널 정보 출력 : psignal(3)

#include < siginfo.h > void psignal (int sig, const char *s); // sig : 정보를 출력할 시그널 // s : 출력할 문자열

- s로 지정한 문자열을 출력한 후 첫 번째 인자인 sig로 지정한 시그널을 가리키는 이름을 붙여 표준 오류로 출력

- 시그널 정보 출력 : strsignal(3)

#include < string.h > char *strsignal (int sig); // sig : 정보를 출력할 시그널

- 인자로 받은 시그널을 가리키는 이름을 문자열로 리턴 (인자로 받은 시그널이 없으면 NULL 리턴)

시그널 블록킹과 해제

- 시그널 불록과 해제 : sighold(3), sigrelse(3)

#include < signal.h > int sighold (int sig); int sigrelse (int sig); // sig : 블록하거나 해제할 시그널

- sighold : 인자로 받은 시그널을 프로세스의 시그널 마스크(Signal mask)에 추가
    - 마스크에 추가된 시그널은 블록되어 해당 시그널을 받지 않음
- sigrelse : 프로세스의 시그널 마스크에서 시그널 해제

- 시그널 집합 블록과 해제 : sigprocmask(2)

#include < signal.h > int sigprocmask (int how, const sigset_t *restrict set, sigset_t *restrict oset);

- how : 시그널을 블록할 것인지, 해제할 것인지 여부. 다음과 같은 값이 올 수 있음
  - SIG_BLOCK : set에 지정한 시그널 집합을 시그널 마스크에 추가
  - SIG_UNBLOCK : set에 지정한 시그널 집합을 시그널 마스크에서 제거
  - SIG_SETMASK : set에 지정한 시그널 집합으로 현재 시그널 마스크를 대체
- set은 블록하거나 해제할 시그널 집합을 가리킴(주소)
- oset은 NULL 또는 이전 설정값이 저장됨

시그널 기다리기

- 시그널 대기 : sigpause(3)

#include < signal.h > int sigpause (int sig);

- 인자로 지정한 시그널을 프로세스의 시그널 마스크에서 제거하고 프로세스가 시그널을 받을 때까지 기다림

- 시그널 기다리기 : sigsuspend(2)

#include < signal.h > int sigsuspend (const sigset_t *set) // set : 기다리려는 시그널을 지정한 시그널 집합의 주소

- 인자로 지정한 시그널 집합에 설정된시그널들로 프로세스의 시그널 마스크를 교체
- 블록되지 않은 시그널이 도착할 대까지 프로세스의 수행을 멈추고 기다림
- 시그널이 도착하면 프로세스의 시그널 마스크는 이전 설정으로 되돌아감

기타 시그널 관련 함수

- 시그널 보내기 : sigsend(2)

#include < signal.h > int sigsend (idtype_t idtype, id_t id, int sig); // idtype : id에 지정한 값의 종류 // id : 시그널을 받을 프로세스나 프로세스 그룹 // sig : 보내려는 시그널

- sig로 지정한 시그널을 id에 지정한 프로세스나 프로세스 그룹에 보냄
- idtype은 id에 지정한 값의 의미를 알려줌
- sigsend 함수가 kill 함수와 다른점은 특정 프로세스 뿐만 아니라 프로세스 그룹 등 시그널을 받을 대상을 다양하게 지정할 수 있음
- idtype은 다음 인자로 사용 할 수 있음

값
의미
P_PID
프로세스 ID가 id인 프로세스에 시그널을 보냄
P_PGID
프로세스 그룹 ID가 id인 모든 프로세스에 시그널을 보냄
P_SID
세션 ID가 id인 모든 프로세스에 시그널을 보냄
P_TASKID
태스크 ID가 id인 모든 프로세스에 시그널을 보냄
P_UID
유효사용자 ID(EUID)가 id인 모든 프로세스에 시그널을 보냄
P_GID
유효 그룹 ID(EGID)가 id인 모든 프로세스에 시그널을 보냄
P_PROJID
프로젝트 ID가 id인 모든 프로세스에 시그널을 보냄
P_CID
스케줄러 클래스 ID가 id인 모든 프로세스에 시그널을 보냄
P_CTID
프로세스 콘트랙트 ID가 id인 모든 프로세스에 시그널을 보냄
P_ALL
id를 무시하고 모든 프로세스에 시그널을 보냄
P_MYID
함수를 호출하는 자신에게 시그널을 보냄

- PID가 0인 프로세스는 시그널을 보내는 대상에서 항상 제외
- P_PID로 직접 지정할때를 제외하고 PID가 1인 프로세스에 시그널을 보내는것도 제외

- 시그널 무시 처리 : sigignore(3)

#include < signal.h > int sigignore (int sig); // sig : 무시할 시그널

- 인자로 지정한 시그널의 처리방법을 SIG_IGN으로 설정
